## Earth Management System

English | [中文](./README.zh-CN.md)

This project is a fictional, planet-scale control and monitoring dashboard built around the concept of “managing Earth”. It focuses on UI/UX and data visualization rather than real-world production data or back-end services.

All UI and interaction patterns are built on top of the open-source component library [`neo-brutal-vue`](https://github.com/i-shl/neo-brutal-vue).  
The entire project was generated by AI to demonstrate how this library can be used in a real-world-style dashboard scenario.

---

### 1. Concept & Scope

- **Purpose**: A central command console for a fictional planet management system  
- **Use cases**: Sci‑fi style big-screen demos, prototypes, UI design practice, data‑viz experiments  
- **Goal**: Provide a cohesive “control center” experience with multiple panels, metrics, and maps

---

### 2. Main Modules

- **Global monitoring dashboard**
  - Fullscreen view with system uptime and current time
  - Key metrics: total population, number of countries, land and ocean area, social stability index, critical alerts, etc.
  - Multiple chart panels for population distribution, resource reserves, environmental health, economic indicators
  - Event & alert feed for critical incidents and pending actions

- **Dashboard view**
  - Summary cards for core metrics
  - Combined charts for population, resources, environment, economy
  - Supports theme switching and a bold neo‑brutalist visual style

- **Geography management**
  - **Continents**: area, population, environment, economy
  - **Countries**: simulated data for population, GDP, resources, environment
  - **Resources**: global natural resources, reserves, distribution and usage

- **Biosphere management**
  - **Species view**: distributions, populations, habitats, conservation status
  - **Humans view**:
    - Cards: total population, active population, average health, age, income, education level
    - Charts: age bands, occupation distribution, health index trends, income segments
    - Table with filtering and search by name, country, occupation, status
    - Entry points for add / detail / edit / delete (some are demo placeholders)

- **Operations & satellites**
  - Satellite view for orbital assets, status, type, and orbit
  - Suitable as a prototype for space activity monitoring and scheduling

---

### 3. Visual Design

- Strong borders and offset shadows in a neo‑brutalist style  
- Grid backgrounds, scanlines, corner frames and badges to create a “control room” feeling  
- Unified styling for tables, cards, charts and text, with high contrast and heavy weights  
- Consistent layouts across management pages to make the system easy to extend

---

### 4. Data & Interactions

- All data is fictional and for demo / learning purposes only  
- Many views show both chart‑level and table‑level information for macro + micro understanding  
- Some buttons and actions are placeholders and do not talk to a real back end  
- Theme and style controls allow adapting the UI to big screens or desktop layouts

### 5. Running the Project (Overview)

> The project follows a typical modern front‑end workflow.  
> Exact commands may vary depending on your environment, so only a generic outline is provided here.

Typical steps:

1. Install a recent Node.js and your preferred package manager  
2. Install dependencies in the project root  
3. Start the dev server and open the local URL in your browser  
4. Run the build command to generate production assets when needed

If you are already familiar with modern front‑end tooling, you can simply follow your usual install / dev / build workflow.
